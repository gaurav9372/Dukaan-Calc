## 2025-02-14 - Semantic Labels for Dynamic Input Groups
**Learning:** In the Dukaan Calculator, when replacing standard browser `<input>` tags with styled `.input-group` wrappers and custom on-screen keyboards, visual `<span class="result-label">` elements break screen reader context. Screen readers cannot properly associate the visually related text with the input field. Additionally, dynamically generated inputs (like those in the fertilizer calculator) lack context without explicit labels.
**Action:** When working with custom input components or wrappers that separate the visual label from the input element, always use semantic `<label for="[id]">` tags instead of `<span>` for static elements. For dynamically generated inputs where unique IDs might be difficult to manage or visual labels don't exist per-input, explicitly inject `aria-label` attributes during DOM creation to ensure accessibility is maintained for assistive technologies.
