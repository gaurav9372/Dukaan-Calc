## 2024-05-24 - DOM Caching for Expensive Lookups
**Learning:** The application updates caret position dynamically on keystrokes using custom DOM-based logic. Continually querying the DOM structure (`closest`, `querySelector`) and forcing layout recalculations (`getComputedStyle` and element dimensions) on every keystroke causes layout thrashing and performance degradation, especially on slower devices. Attaching cached values and DOM element references directly to the input element nodes (e.g. `input._cachedGroup`) significantly improves performance but requires careful lifecycle management (cache invalidation on resize).
**Action:** Implement a DOM caching pattern on element nodes for hot code paths like typing interactions. Ensure explicit `undefined` checks (e.g., `if (input._cachedGroup === undefined)`) to safely identify uninitialized caches without tripping over falsy values. Implement an `invalidateCaches` function connected to `resize` and `orientationchange` events to ensure cached dimensions accurately reflect the layout.
