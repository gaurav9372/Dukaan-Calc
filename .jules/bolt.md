## 2024-05-20 - Synchronous DOM Layout Thrashing on High-Frequency Events
**Learning:** Calling `window.getComputedStyle` synchronously during high-frequency events like text input (e.g., updating a caret position) interleaves DOM reads with DOM writes across the app, causing layout thrashing and severe performance degradation. This is particularly problematic in a codebase utilizing a custom virtual caret dependent on accurate layout dimensions.
**Action:** Implement DOM element and static style caching (e.g., attaching properties like `_cachedFont` directly to the elements) to skip expensive `getComputedStyle` and `closest` lookups. Use an explicit `undefined` check (`if (el._cached === undefined)`) and ensure an `invalidateCaches` function is triggered on `resize` and `orientationchange` events to keep layout data fresh.
