## 2024-05-24 - DOM Caching to Prevent Layout Thrashing in High-Frequency Events
**Learning:** In highly interactive components with custom keystroke handling, frequently calling `getComputedStyle` and `closest` during each key press (e.g., in functions like `updateCaret`) causes severe layout thrashing and blocks the main thread.
**Action:** Implemented a DOM caching pattern where element references (like parent `.input-group`) and static computed styles (like `.paddingLeft` and `font`) are attached directly to the DOM nodes (e.g., `input._cachedGroup`). Explicit `undefined` checks are used to safely initialize them. A global `invalidateCaches` function is bound to `resize` and `orientationchange` events to safely clear the cache and allow recalculation upon layout changes.
