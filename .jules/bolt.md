## 2024-04-03 - Layout Thrashing with Custom On-Screen Keyboard Repeat
**Learning:** The custom on-screen keyboard repeats keys every 70ms when held down. This rapid firing caused continuous synchronous layout recalculations (layout thrashing) because `updateCaret` and `placeCaretFromEvent` repeatedly called `window.getComputedStyle(input)` and read layout properties, significantly impacting performance and battery life.
**Action:** Implemented a DOM caching pattern by storing static computed styles (e.g., font, padding, and element references) directly on the input DOM elements (`input._cachedGroup`, `input._cachedFont`, etc.) and using an `invalidateCaches` function tied to `resize` and `orientationchange` events to safely clear them when necessary. Always cache expensive layout reads for rapidly recurring UI events.
